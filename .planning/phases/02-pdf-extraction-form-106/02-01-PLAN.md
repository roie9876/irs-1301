<task type="auto">
  <name>Task 2: LLM extraction function and documents router</name>
  <files>backend/app/services/llm.py, backend/app/routers/documents.py</files>
  <action>
1. Add `extract_form106_data(raw_text: str) -> dict` to `backend/app/services/llm.py` per D-02 and D-09:
   - Define `FORM_106_EXTRACTION_PROMPT` as a module-level constant. The prompt specifies the 12 fields from D-07 with Hebrew and English names, Form 1301 field mappings, and instructs the LLM to return JSON with `{"field": {"value": ..., "confidence": 0.0-1.0}}`. Fields not found → `value: null, confidence: 0`. The prompt MUST contain the word "JSON" per Pitfall 4.
   - Function loads .env (same pattern as test_connection), builds LiteLLM kwargs with `response_format={"type": "json_object"}`, calls `await litellm.acompletion(**kwargs)`, parses response JSON.
   - Raises `ValueError("LLM not configured")` if provider/model/api_key missing.

2. Create `backend/app/routers/documents.py` per D-03, D-04, D-10, D-11:
   - Define `DOCUMENTS_DIR = Path(__file__).resolve().parent.parent.parent.parent / "user_data" / "documents"` (project root / user_data / documents — per Pitfall 3, uvicorn runs from backend/).
   - `POST /documents/upload`: Accepts `files: List[UploadFile] = File(...)`. For each file:
     - Validate: filename ends with `.pdf` — else error "הקובץ אינו PDF תקין" per D-10
     - Generate `doc_id = uuid.uuid4().hex[:8]`, save as `{doc_id}_{filename}` in DOCUMENTS_DIR
     - Call `extract_text_from_pdf(str(file_path))` — if empty text, error "לא ניתן לחלץ טקסט מהקובץ"
     - Call `await extract_form106_data(raw_text)` — parse result into `Form106Extraction`
     - Save JSON sidecar: `{safe_filename}.106.json` with doc_id, original_filename, extracted data, user_corrected=False. Use `json.dumps(..., ensure_ascii=False, indent=2)` per Pitfall 6.
     - Return `UploadResponse` with results array.
     - Wrap extraction in try/except — on error, append error result `"שגיאה בחילוץ: {str(e)}"` and continue to next file.
   - `GET /documents`: Scan DOCUMENTS_DIR for `*.106.json` files, parse each, return `DocumentListResponse`.
   - `PUT /documents/{doc_id}`: Accept `UpdateFieldsRequest`, find the JSON sidecar by doc_id prefix, update specified fields, set `user_corrected: true`, save back. Return updated `DocumentInfo`. 404 if not found.

   Do NOT wire the router in main.py — that happens in Plan 02-03.
