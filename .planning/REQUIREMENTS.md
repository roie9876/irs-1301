# Requirements: עוזר דוח שנתי 1301

**Defined:** 2026-04-10
**Core Value:** המשתמש מבין מה למלא בכל שדה בטופס 1301 ומגיש בעצמו בלי עזרה חיצונית

## v1 Requirements

### הגדרת LLM

- [ ] **LLM-01**: המשתמש בוחר ספק LLM מתוך: OpenAI, Azure OpenAI, Google Gemini, Anthropic Claude
- [ ] **LLM-02**: המשתמש מזין API Key אישי שנשמר מקומית בקובץ .env
- [ ] **LLM-03**: המשתמש בוחר מודל ספציפי מתוך הספק (gpt-4o, claude-sonnet, gemini-2.0-flash וכו')
- [ ] **LLM-04**: המערכת מוודאת שה-API Key תקף בעת הגדרה (בדיקת חיבור)
- [ ] **LLM-05**: הגדרות ספק ומודל נשמרות מקומית ולא צריך להזין כל פעם

### עיבוד מסמכים

- [ ] **DOC-01**: המשתמש מעלה טופס 106 (PDF דיגיטלי) — חילוץ: הכנסה ברוטו, מס שנוכה, הפרשות פנסיה/קה"ש, ימי עבודה
- [ ] **DOC-02**: המשתמש מעלה מספר טופסי 106 (ריבוי מעסיקים) — איחוד נתונים
- [ ] **DOC-03**: המשתמש מעלה טופס 867 (רווחי הון) — חילוץ: רווחים, הפסדים, דיבידנדים, מס שנוכה
- [ ] **DOC-04**: המשתמש מעלה דוח RSU שנתי מאקסלס (PDF) — חילוץ: עסקאות מכירה, ניכוי מס במקור
- [ ] **DOC-05**: המשתמש מעלה אישורי תרומות (סעיף 46) — חילוץ: סכום, שם עמותה
- [ ] **DOC-06**: המשתמש מעלה אישורי ביטוח חיים/פנסיה — חילוץ: סכומי הפקדה שנתיים
- [ ] **DOC-07**: המשתמש מעלה קבלות הוצאות רפואיות — חילוץ: סכום, תאריך
- [ ] **DOC-08**: המשתמש מעלה אישורי משכנתא — חילוץ: ריבית שנתית ששולמה
- [ ] **DOC-09**: המשתמש רואה את הנתונים שחולצו מכל מסמך ויכול לתקן ידנית
- [ ] **DOC-10**: כל המסמכים והנתונים נשמרים בתיקייה מקומית בלבד

### RAG — מנוע ידע

- [ ] **RAG-01**: מסמכי ההנחיות של רשות המסים (PDF) מאונדקסים לפי שנת מס
- [ ] **RAG-02**: כל שנת מס = אינדקס ChromaDB נפרד (collection) — מונע ערבוב בין שנים
- [ ] **RAG-03**: כשהמשתמש ממלא שדה, המערכת שולפת הסבר רלוונטי ממסמך ההנחיות של אותה שנה
- [ ] **RAG-04**: אינדקס מאוחסן מקומית (ChromaDB)

### Wizard — מילוי טופס 1301

- [ ] **WIZ-01**: המשתמש בוחר שנת מס (2021-2025)
- [ ] **WIZ-02**: שאלון מקדים: שכיר? נשוי? ילדים? נכס להשכרה? מניות? — קובע חלקים רלוונטיים
- [ ] **WIZ-03**: הטופס מוצג כרצף שלבים, חלקים לא רלוונטיים מוסתרים
- [ ] **WIZ-04**: לכל שדה: שם, הסבר מותאם מ-LLM (על בסיס RAG + מצב המשתמש), ערך מוצע (אם חולץ ממסמך)
- [ ] **WIZ-05**: המשתמש יכול לשאול שאלה חופשית על כל שדה ("מה זה? האם זה רלוונטי לי?")
- [ ] **WIZ-06**: חישוב נקודות זיכוי — אישיות, ילדים, יישובים, נכות
- [ ] **WIZ-07**: שמירת התקדמות — המשתמש יכול לסגור ולחזור מאוחר יותר

### דוח משותף (בן/בת זוג)

- [ ] **SPO-01**: המשתמש מסמן שהדוח משותף עם בן/בת זוג
- [ ] **SPO-02**: הזנת נתונים נפרדת לכל בן זוג (הכנסות, ניכויים, מסמכים)
- [ ] **SPO-03**: הצגת עמודות נפרדות בטופס (נישום / בן זוג) כמו בטופס 1301 המקורי

### צ'אט חופשי

- [ ] **CHAT-01**: ממשק צ'אט לשאלות חופשיות בנושא מס הכנסה
- [ ] **CHAT-02**: הצ'אט מודע להקשר — שנת מס, מסמכים שהועלו, שדה נוכחי
- [ ] **CHAT-03**: תשובות מבוססות RAG (מסמך ההנחיות) ולא רק ידע כללי של המודל

### הכנסות נוספות

- [ ] **INC-01**: הכנסות מהשכרת דירה — בחירת מסלול מיסוי (פטור/10%/שולי)
- [ ] **INC-02**: הכנסות מריבית (פיקדונות, דמי פיקדון)
- [ ] **INC-03**: החזר מס על משכנתא

### תשתית

- [ ] **INF-01**: ממשק עברית מלא RTL
